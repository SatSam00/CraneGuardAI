# Contributing to CraneGuard AI

Thank you for your interest in contributing to CraneGuard AI! We welcome community contributions to help make industrial environments safer.

## 🛠️ Development Workflow

1.  **Fork the Repository**: Create your own copy of the project.
2.  **Create a Feature Branch**: `git checkout -b feature/amazing-feature`.
3.  **Install Dependencies**: Follow the instructions in [README.md](./README.md).
4.  **Implement Changes**: Follow our coding standards.
5.  **Test Locally**: Ensure the backend and frontend modules interact correctly.
6.  **Submit a Pull Request**: Describe your changes in detail and link any relevant issues.

## 📝 Coding Standards

### Backend (Python/FastAPI)
*   **PEP 8**: Follow standard Python styling.
*   **Type Hinting**: Use type hints for all function arguments and return values.
*   **Docstrings**: All publicly accessible functions and classes should have Google-style docstrings.
*   **Async/Await**: Prefer asynchronous operations where possible (e.g., DB calls, HTTP requests).

### Frontend (React/Vite)
*   **Component Architecture**: Keep components small and focused.
*   **TailwindCSS**: Use Tailwind utility classes for styling. Avoid inline styles or complex CSS files.
*   **ES6+**: Use modern JavaScript features (arrow functions, destructuring, template literals).
*   **Props Validation**: Ensure correct prop types are passed to components.

## 🛠️ Reporting Bugs

If you find a bug, please open an issue with the following details:
1.  **Summary**: A clear and concise description of the bug.
2.  **Steps to Reproduce**: How can we trigger the bug?
3.  **Expected Behavior**: What should have happened?
4.  **Actual Behavior**: What actually happened (include logs or screenshots if possible).

## 🚀 Feature Requests

We are always looking for new ideas! Please open an issue with the "feature-request" label to discuss any enhancements you'd like to see.
